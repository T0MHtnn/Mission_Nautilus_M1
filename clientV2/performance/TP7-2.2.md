# Rapport de Performance - TP7 - 2.2 (Canvas Bulles)

## 1. Fonctionnalité ajoutée
Ajout d'un canvas 2D en sur-impression sur la carte Leaflet, affichant
une animation continue de bulles remontantes et un fond bleuté simulant
un environnement sous-marin. Le canvas est implémenté en composant Vue
via `waterOverlay.js` et utilise `requestAnimationFrame` pour la boucle
d'animation. L'interaction avec la carte reste possible grâce à
`pointer-events: none` sur le canvas.

## 2. Métriques Lighthouse (Mobile)

| Métrique | Valeur |
| :--- | :--- |
| **Score Global** | 47 / 100 |
| **First Contentful Paint** | 18.8 s |
| **Largest Contentful Paint** | 24.2 s |
| **Total Blocking Time** | 360 ms |
| **Cumulative Layout Shift** | 0 |
| **Speed Index** | 18.8 s |

## 3. Analyse

L'ajout du canvas a eu un impact négatif mesurable sur le TBT :
celui-ci est passé de 100 ms (v2.1) à **360 ms**, ce qui confirme que
la boucle `requestAnimationFrame` sollicite davantage le thread
principal. Le score global passe de 55 à 47.

Les principaux problèmes identifiés sont :
- **Minify JavaScript** : gain potentiel de 2 966 KiB (code non minifié)
- **Reduce unused JavaScript** : gain potentiel de 1 196 KiB
- **Minimize main-thread work** : 6.8 s de travail sur le thread principal
- **Reduce JavaScript execution time** : 3.5 s d'exécution JS

> Ces résultats justifient le passage au **Web Worker** (partie 4) pour
> déplacer le calcul des bulles hors du thread principal.
