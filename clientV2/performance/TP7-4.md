# Rapport de Performance - TP7 - 4 (Web Worker + OffscreenCanvas)

## 1. Optimisation effectuée

### Déplacement du rendu des bulles dans un Web Worker
Le calcul de l'animation des bulles (`waterOverlay.js`) a été séparé en
deux parties :

- **`waterOverlay.js`** (thread principal) : initialisation du canvas
  dans le DOM, transfert de l'`OffscreenCanvas` au worker via
  `postMessage`, gestion du redimensionnement.
- **`waterWorker.js`** (worker dédié) : reçoit l'`OffscreenCanvas`,
  effectue tous les calculs de rendu (bulles, ondulations, gradient) en
  tâche de fond via `requestAnimationFrame`.

Le canvas est transféré avec `transferControlToOffscreen()` et passé
comme objet transférable au worker, ce qui évite toute copie de données.

## 3. Analyse

Les métriques Lighthouse restent stables par rapport à la v3. Le léger
écart sur le TBT (+10ms) est dû au coût d'initialisation du worker
(création du thread, transfert de l'OffscreenCanvas).

Le gain réel du Web Worker n'est pas visible dans Lighthouse car
celui-ci mesure le chargement initial — le bénéfice se manifeste
surtout en **runtime** : le thread principal n'est plus bloqué par les
calculs de rendu des bulles à 60fps, ce qui améliore la réactivité
de la carte pendant le jeu.

Le thread principal passe de 2.6s à **2.3s** de travail, confirmant
le déchargement effectif des calculs vers le worker.
