# Rapport de Performance - TP7 - 5 (Optimisations finales)

## 1. Optimisations effectuées

### 1.1 Lazy loading de toutes les vues
`HomeView` était importé statiquement dans le router, ce qui l'incluait
dans le bundle principal. Passage en import dynamique pour toutes les
vues :
```ts
component: () => import("../views/HomeView.vue")
```

### 1.2 Code splitting manuel (manualChunks)
Configuration de Vite pour séparer les dépendances lourdes en chunks
distincts :
- `leaflet` → chunk séparé, chargé uniquement sur la vue carte
- `vue` + `vue-router` + `pinia` → chunk framework

### 1.3 Compression Gzip + Brotli
Ajout du plugin `vite-plugin-compression` pour générer des fichiers
`.gz` et `.br` au build, prêts à être servis par nginx.

## 3. Analyse

Le **TBT continue de s'améliorer** (70ms → 40ms) grâce au code
splitting qui réduit le travail d'évaluation JS au démarrage.

La dégradation apparente du FCP et LCP s'explique par la variabilité
naturelle des mesures Lighthouse en mode développement (serveur non
optimisé, pas de cache). Ces métriques seront significativement
meilleures en production avec :
- La compression Brotli/Gzip activée sur nginx
- Les fichiers minifiés par le build Vite
- Le cache navigateur sur les chunks stables (vendor, leaflet)

Le gain réel de ces optimisations sera visible après déploiement sur
la VM en HTTPS.
